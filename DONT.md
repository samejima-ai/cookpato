# スコープ外定義

以下はこのプロジェクトのスコープ外とする。
**意図的に排除している**。AI能力の問題ではなく、「この機能があることがユーザー体験を悪化させる」と判断したもの。

---

## 意識的に排除する機能（既存献立アプリにあるが本アプリでは実装しない）

| 排除する機能 | 排除理由 |
|---|---|
| カテゴリ分け（主菜/副菜/汁物等） | 親子丼のように複数カテゴリにまたがるものを1分類に強制できない |
| 料理名のマスター登録・統一 | 表記ゆれを本人が気にしないため、強制すると手数が増える |
| レシピ登録・表示 | 既に把握している料理を作る用途であり、レシピは不要 |
| 栄養計算 | 計算に必要な入力が増えることで手軽さを失う |
| 買い物リスト自動生成 | ユーザーが手動で把握しているため不要 |
| 冷蔵庫の全在庫管理 | 既存の記憶方式で回っているため不要（作り置きストックは別機能として実装） |
| 家族・パートナー共有 | 単一ユーザー前提 |
| SNS共有 | 個人管理用途 |
| 写真添付 | 手軽さを損なう |
| 通知・リマインダー | 本人の記憶で運用できている |

---

## 実装が複雑になるため除外（AI能力の現状の限界）

- 創造的UXデザイン（アニメーション、独自UIコンポーネント）
- 複雑な状態管理UI（ドラッグ&ドロップでの献立並び替え、ピンチズーム等）
- パフォーマンス最適化（ただし無限スクロールの軽快さは機能要件として別途満たす）
- 多言語対応（日本語のみ）
- アクセシビリティの高度な対応（VoiceOver最適化等）

---

## 明示的に行わない通信・同期系

- クラウド同期・マルチデバイス同期
- サーバーAPIとのデータ通信（データは完全ローカル）
- バックアップ・エクスポート機能
- プッシュ通知
- アカウント管理・ログイン機能

※ PWA（Service Worker + manifest）は「ホーム画面追加」と「オフライン動作」の手段として**採用する**。
ただし通信を伴う用途（プッシュ通知、バックグラウンド同期等）には使わない。

---

## 将来スコープに入る可能性があるもの

以下は現時点で除外しているが、本人の要望次第で将来的に追加候補となる：

- 週次サマリー表示（「今週魚は何回？」等）
- タグ機能（本人が任意で付けられる、強制はしない）
- 複数端末同期

ただし追加する場合も、「メモ帳の手軽さを損なわない」を最優先とする。
「あったら便利」レベルでは実装しない。

---

## スコープ外の基本原則

> **使わない機能が画面にあること自体がユーザー体験を悪化させる。**

既存の献立アプリをやめた最大の理由は、機能過多による操作負担だった。
本アプリの競合は市販の献立アプリではなく「メモ帳」である。
メモ帳より手軽でなければ存在価値がない。
