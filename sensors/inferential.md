# 推論的センサー

LLM による確率的判定。計算的センサー全通過後に実行する。
「仕様に合う・動く・使える」の3観点で自己評価する。

---

## 1. 仕様合致チェック（SPEC.md との照合）

### カレンダー（無限スクロール）
- [ ] 起動時に当日が画面中央付近に来るよう自動スクロールされるか
- [ ] 上下スクロールで過去/未来の日付に連続的に移動できるか
- [ ] 月切替ボタンが**ない**か（スクロールのみで移動する設計か）
- [ ] 各行が「M月D日（曜）」形式で表示されるか
- [ ] 月の区切りに月ヘッダーが挿入されるか
- [ ] 日曜の行・当日の行が視覚的に区別されるか

### フリー入力
- [ ] 1日に1つの入力欄のみか（主菜/副菜の区別UIがないこと）
- [ ] 保存/登録ボタンが**ない**か
- [ ] 入力後、再読み込みしても即時保存されているか
- [ ] 表記正規化されず、入力したままが保存されるか
- [ ] 1行=1品として扱われるか
- [ ] 可視範囲付近の日付行は、タップ不要で直接 textarea 入力できるか（2026-04 改訂）
- [ ] 連続する複数日への書き込みが、編集モード切替なしで完結するか
- [ ] 範囲外（遠い日付）はタップ進入方式が維持されているか（DOMコスト抑制）

### 完了トグル（品単位）
- [ ] 各行（品）にトグルUIが配置されているか
- [ ] タップで即切替されるか
- [ ] 完了行が打ち消し線等で区別されるか
- [ ] 日単位のまとめトグルが**ない**か
- [ ] 並び替えUIが**ない**か（追加・削除のみ）
- [ ] トグルのヒット領域が44×44pxを上回り、行左側1/3以上を占めるか（2026-04 改訂）
- [ ] 完了状態の視覚フィードバックが3点併用（打ち消し線＋文字色＋背景色）で表現されているか
- [ ] 触覚フィードバック（Haptic / vibrate）の発火が試行されているか（非対応端末ではエラーにならないこと）
- [ ] 300ms以内の連続タップが1回として扱われるか（チャタリング防止）
- [ ] トグルタップで textarea にフォーカスが移らないか（領域分離）

### 過去履歴検索
- [ ] 検索欄が空のとき、検索結果が**表示されない**か（カレンダーが見える）
- [ ] キーワード入力時のみ結果オーバーレイが出るか
- [ ] ひらがな/カタカナが相互マッチするか（「豚バラ」⇔「ぶたばら」）
- [ ] 全角/半角、大文字/小文字が吸収されるか
- [ ] 結果が日付降順で最大20件表示されるか
- [ ] 結果タップで該当日にスクロールし、パネルが閉じるか

### ストックリスト
- [ ] 画面下部に常時表示されるか
- [ ] 折りたたみ可能か、デフォルト展開か
- [ ] 追加・削除ができるか
- [ ] アプリ再起動後も内容が保持されるか
- [ ] カレンダーとストックリストが同一画面に両立しているか
- [ ] 削除がタップ単独で完結するか（スワイプ・長押しが**ない**か）（2026-04 改訂）
- [ ] 削除ボタンのタップ領域が44×44px以上か

- [ ] 料理名マスター・統一機能が**ない**か
- [ ] レシピ登録・表示機能が**ない**か
- [ ] 栄養計算機能が**ない**か
- [ ] 買い物リスト機能が**ない**か
- [ ] 写真添付機能が**ない**か
- [ ] 通知・リマインダー機能が**ない**か
- [ ] ログイン・アカウント機能が**ない**か
- [ ] サーバー通信コード（fetch/axios/WebSocket）がコード全体で**1箇所も存在しない**か
- [ ] Analytics/Sentry 等のトラッキングコードが**ない**か

## 3. 「動く」チェック

- [ ] `npm run build` が成功する
- [ ] `npm run preview` でローカルサーバーが立ち上がる
- [ ] iPhone 11 相当のビューポート（375×812）で崩れずに表示される
- [ ] Service Worker が登録され、オフラインでも起動する
- [ ] Web Manifest が出力され、「ホーム画面に追加」時のアイコン・タイトルが正しい
- [ ] JavaScript エラーがコンソールに出ない

---

## 4. 「使える」チェック（ユーザーシナリオ）

以下のシナリオを一通り通せること：

1. アプリを開く → 当日の行がすぐ見える位置にある
2. 今日の行をタップし、「豚バラ大根」「サラダ」と改行入力する → 即保存される
3. 上へスクロールして過去の日を確認 → 軽快に動く
4. 検索欄に「ぶたばら」と入力 → 「豚バラ大根」を含む日がヒット表示される
5. 検索結果をタップ → 該当日にスクロールし、検索パネルが閉じる
6. 検索欄をクリア → カレンダーに戻る
7. 画面下部のストックリストに「グラタンのもと 1個」を追加 → 保持される
8. 「豚バラ大根」の行のトグルをタップ → 打ち消し線が付く
9. アプリを完全にリロード → 入力・完了状態・ストックが全て保持されている
10. 機内モード（オフライン）でアプリを開き直す → 正常に起動する

---

## 判定基準

- 全項目クリア → 献上可能
- 未クリア項目あり → 自力修正を試みる（同一問題最大3回）→ 不可ならフィードバックレポートに記載して献上
- 「スコープ侵食チェック」に1項目でも該当 → 重大。必ず修正してから献上

---

## 判定が微妙なケース

- 「軽快」「即時」などの主観的判断基準 → iPhone 11 相当でのレスポンスが体感200ms以内を目安とする
- 「視覚的に区別」 → 目視で識別可能であればOK。デザインの美しさは評価対象外
